# Completed Fixes & Features

1. **Context-pin drag crash fix (v1)** — `EnsureHiddenK2CompatGraph()` creates `__HiddenK2CompatGraph__` (direct child of `UBLBTStateMachineBlueprint`) with `UEdGraphSchema_K2` schema, preventing `FBlueprintNodeTemplateCache::FindCompatibleGraph()` from crashing when dragging pins.
2. **Custom editor layout** — Replaced `FBlueprintEditorApplicationMode` with `FBLBTStateMachineApplicationMode`. Layout: left sidebar (States 40%, Transitions 35%, Details 25%) + center Document area.
3. **Glass/cyan editor styling** — Added `BLBTEditorStyle.h`, wrapped sidebar panels in double-layer `SBorder` with cyan border/background colors, and set `AppearanceInfo.CornerText = "BLBT STATE MACHINE"` with a custom badge brush.
4. **Load-time compiler crash fix** — Fixed `FKismetCompilerContext::IsNodePure` ensure/crash by temporarily removing `StateMachineGraph` from `UbergraphPages` during `FBLBTStateMachineCompilerContext::CreateFunctionList()`, then restoring it.
5. **Transition graph naming** — `BLBTGraphNode_Transition::CreateConnections` renames the transition and its `BoundGraph` to `EvaluateCondition_StateA_StateB`. `BLBTTransitionGraphSchema::GetGraphDisplayInformation` shows tab title as `StateA -> StateB`.
6. **Toolbar fix** — Compile/Save buttons moved to the top toolbar via `FBlueprintEditorToolbar::AddCompileToolbar` + `AddBlueprintGlobalOptionsToolbar` in `FBLBTStateMachineApplicationMode` constructor. Non-functional sidebar Compile button removed.
7. **Main viewport close prevention** — Implemented `FixStateMachineGraphTabClose()` which scans `DocumentManager->GetAllDocumentTabs()`, finds the tab containing the `StateMachineGraph` `SGraphEditor`, and calls `SetCanCloseTab(false)`. Called on init and `OnActiveTabChanged`.
8. **Interactive sidebar panels** — `FBLBTStatesSummoner` and `FBLBTTransitionsSummoner` now use `SListView`. Single-click selects/jumps to the node in the graph; double-click on a State opens its `BoundGraph`; double-click on a Transition opens its condition graph.
9. **Details panel editable** — `FBLBTDetailsSummoner` now reuses the editor's built-in `SKismetInspector` (like engine `FSelectionDetailsSummoner`), so selecting a node allows editing properties.
10. **Sidebar auto-refresh** — `FBLBTStateMachineBlueprintEditor` binds `StateMachineGraph->OnGraphChanged` and overrides `OnBlueprintChangedImpl`. Adding/deleting States/Transitions or compiling now automatically refreshes the sidebar lists.
11. **Entry State visual identity** — Entry State node title is rendered in bright green (`FLinearColor(0.0f, 0.95f, 0.35f)`). Custom `SBLBTGraphPin` also tints its pins green when the owning State is the Entry State.
12. **Entry State auto-management** — Newly created Blueprints automatically sync the default "Start" node's Guid to `EntryStateGuid`. Adding the first State auto-sets it as Entry if none exists. Deleting the Entry State auto-transfers Entry to another remaining State.
13. **Context-pin drag crash fix (v2)** — Transition `BoundGraph`s were being picked up by `FindCompatibleGraph` because their Outer was the Blueprint. Changed `BLBTGraphNode_Transition::CreateBoundGraph()` to create the graph with Outer = `this` (the Transition node), so only `__HiddenK2CompatGraph__` is found.
14. **PLAIBehaviorComponent integration** — `PLAIBehaviorComponent` now respects `FMonsterInfo::StateMachineBlueprint`. If configured, it disables Root Tree / DynamicTree and exposes `RunStateMachine()` for explicit startup, plus `IsUsingStateMachine()` / `IsStateMachineRunning()` query APIs.
